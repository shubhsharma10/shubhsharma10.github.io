This is my first post. I have been thinking of building a database from scratch for a long time(~ 4 years). After LLMs I think for the first time I feel one can learn anything if there is intent. Many people who are much smarter than me have said Action is more powerful than Talent.

So I am also starting my journey. Idea is to write each component by myself without using any LLM tools. I will use LLM tools to guide me. Components I will build are following:

  1. Append-only KV store with indexing, crash recovery of indexing (Bitcask)
  2. Persistent KV Store with SS Table, Background Compaction and Tombstone for deletes (LevelDB, RocksDB)
  3. B+ Tree Index with Range scans, Ordered iteration, Page splitting, Disk-backed nodes and Visual debugger for tree (SQLite)
  4. ACID with WAL with MVCC (Postgres)
  5. Query engine with Parser, Planner, Optimizer, Execution engine
  6. Replicated KV Store with Raft (etcd and Cockroach DB)
  7. OLAP / Search Engine / Streaming DB

My hope is that by the end of this I will have understood what databases are at a fundamental level. I have read multiple books. But it's always been very hand-wavy. Unless you build something from scratch you don't understand complexities involved in it. Well starting anyway :)
